Architecture

The system follows an action-driven loop: the LLM returns structured JSON with an array of actions, and the runtime executes them sequentially against the conversation state.

Runtime

Orchestrates the full loop: receives user input, builds prompts, calls LLM, parses actions, executes them, persists state. Knows nothing about the CLI.

Config (Pydantic)

Single Config.load(path) validates and loads the entire agent definition. All behavior flows from it.

Structured Output

LLM responses constrained via response_format JSON Schema with "strict": true. No parsing guesswork.

Data Flow

User Input → Build Prompt → LLM Call → Parse Actions → Confidence Gate → Execute → Display → Persist State

Key Design Decisions

  • Actions self-execute on state, return str | None — Only SendMessage returns displayable text. Others silently mutate state. The runtime doesn't need to know what each action does.
  • Buffered display output — All action logs print as they run, but agent messages display only after all actions complete. Prevents interleaved output.
  • Confidence gating at runtime — Defense-in-depth: the prompt tells the LLM not to collect low-confidence data, but the runtime also rejects collect_field below the configured threshold.
  • Structured Output with strict JSON Schema — response_format enforces the exact shape at the API level. Graceful fallback if truncated.
  • Steps managed by the LLM, not the runtime — Steps are high-level flow markers. The LLM infers transitions from context and updates them via update_state.
  • System prompt as interpolated markdown — Template uses headings, bold, tables for better LLM comprehension. Config values and state injected at call time.

Action Routing

The LLM returns {"actions": [...]} — each with a type and payload. The runtime routes them:

send_message

Display text to user. Must be last action. Returns the content string.

collect_field

Store a field value with confidence score. Gated by global confidence_threshold.

update_state

Update conversation flow steps (greeting → collecting → escalation).

escalate

Hand off to human agent. Sets escalated=true with a reason.

Design Patterns

Command

Each Action encapsulates an operation. Runtime iterates and executes without knowing the implementation.

Repository

StateStore — CRUD-only data access. Memory for tests, file-based for production.

Strategy

LLMClient — swap OpenRouter for a test double or another provider.

Observer

OutputHandler receives real-time callbacks. Status bar updates after every single action.
